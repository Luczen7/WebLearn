1 Introduction à JavaScript

JavaScript est le langage de programmation du web. Créé en 1995, il est aujourd'hui indispensable pour créer des sites web interactifs et des applications web modernes.

1.1 Qu'est-ce que JavaScript ?

JavaScript est un langage de programmation :

  • Interprété : exécuté ligne par ligne par le navigateur
  • Orienté objet : basé sur des objets et des prototypes
  • Événementiel : réagit aux actions de l'utilisateur
  • Asynchrone : capable d'effectuer plusieurs tâches simultanément

1.2 Où écrire du JavaScript ?

Vous pouvez inclure JavaScript de trois façons :

HTML - Inclusion JS 
<!-- 1. Inline (à éviter) -->
<button onclick="alert('Hello')">Cliquez</button>

<!-- 2. Interne -->
<script>
  console.log('Hello World');
</script>

<!-- 3. Externe (recommandé) -->
<script src="app.js" defer></script>
<!-- defer : exécute après le parsing HTML -->
Bonne pratique

Toujours placer les balises <script> en bas du body ou utiliser l'attribut defer dans le head. Cela évite de bloquer le rendu de la page.

2 Variables et Types de Données

2.1 Déclaration de variables

JavaScript propose trois mots-clés pour déclarer des variables :

JavaScript 
// var - fonction-scopé, peut être redéclaré (éviter)
var nom = 'Alice';

// let - bloc-scopé, peut être réassigné
let age = 25;
age = 26; // OK

// const - bloc-scopé, ne peut pas être réassigné
const PI = 3.14159;
// PI = 3.14; // ERREUR !

// const avec objets : la référence est constante
const user = { name: 'Bob' };
user.name = 'Charlie'; // OK !
// user = {}; // ERREUR

3 Fonctions en JavaScript

3.1 Déclaration de fonctions

JavaScript - Fonctions 
// Déclaration de fonction (hoisted)
function saluer(nom) {
  return `Bonjour ${nom} !`;
}

// Expression de fonction
const addition = function(a, b) {
  return a + b;
};

// Arrow function (ES6) - syntaxe concise
const multiplication = (a, b) => a * b;

// Arrow function avec bloc
const division = (a, b) => {
  if (b === 0) throw new Error('Division par zéro');
  return a / b;
};

// Paramètres par défaut (ES6)
function creerUser(nom, role = 'user') {
  return { nom, role };
}

// Rest parameters (ES6)
function somme(...nombres) {
  return nombres.reduce((acc, n) => acc + n, 0);
}
somme(1, 2, 3, 4); // 10

3.2 Fonctions de callback et Higher-Order Functions

Les fonctions sont des citoyens de première classe en JavaScript :

JavaScript - HOF 
const nombres = [1, 2, 3, 4, 5];

// map : transformer chaque élément
const doubles = nombres.map(n => n * 2);
// [2, 4, 6, 8, 10]

// filter : garder certains éléments
const pairs = nombres.filter(n => n % 2 === 0);
// [2, 4]

// reduce : réduire à une seule valeur
const total = nombres.reduce((acc, n) => acc + n, 0);
// 15

// find : trouver le premier élément
const premierPair = nombres.find(n => n % 2 === 0);
// 2

// some / every : tests booléens
const aUnPair = nombres.some(n => n % 2 === 0); // true
const tousPairs = nombres.every(n => n % 2 === 0); // false
Astuce

Préférez les méthodes fonctionnelles (map, filter, reduce) aux boucles impératives (for, while). Le code est plus lisible, déclaratif et moins sujet aux erreurs.

4 Objets et Tableaux Avancés

4.1 Destructuring (ES6)

Le destructuring permet d'extraire des valeurs d'objets et de tableaux de manière concise :

JavaScript - Destructuring 
// Destructuring d'objet
const user = { nom: 'Alice', age: 30, ville: 'Paris' };
const { nom, age } = user;

// Avec renommage
const { nom: prenom, ville = 'Inconnue' } = user;

// Destructuring de tableau
const [premier, deuxieme, ...reste] = [10, 20, 30, 40, 50];
// premier = 10, deuxieme = 20, reste = [30, 40, 50]

// Destructuring dans les paramètres
function afficherUser({ nom, age }) {
  console.log(`${nom} a ${age} ans`);
}

// Nested destructuring
const data = { user: { profile: { email: 'test@test.com' } } };
const { user: { profile: { email } } } = data;

4.2 Spread et Rest operators

JavaScript - Spread/Rest 
// Spread sur tableaux
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5, 6]; // [1, 2, 3, 4, 5, 6]

// Spread sur objets (shallow copy)
const user1 = { nom: 'Bob', age: 25 };
const user2 = { ...user1, ville: 'Lyon', age: 26 };
// { nom: 'Bob', age: 26, ville: 'Lyon' }

// Fusion de tableaux/objets
const merged = { ...defaults, ...options };

// Clonage d'objet
const clone = { ...original };
const deepClone = JSON.parse(JSON.stringify(original)); // Deep copy simple

4.3 Méthodes modernes des tableaux

JavaScript - Tableaux 
const fruits = ['pomme', 'banane', 'orange'];

// includes (ES2016)
fruits.includes('banane'); // true

// flat / flatMap (ES2019)
const nested = [1, [2, 3], [4, [5, 6]]];
nested.flat(2); // [1, 2, 3, 4, 5, 6]

// at (ES2022) - index négatif supporté
fruits.at(-1); // 'orange' (dernier élément)

// toSorted, toReversed, toSpliced (ES2023) - immutables
const sorted = fruits.toSorted(); // Nouveau tableau trié
// fruits reste inchangé !

5 Manipulation du DOM

Le DOM (Document Object Model) est une représentation en mémoire de la structure HTML d'une page. JavaScript permet de le manipuler dynamiquement.

5.1 Sélection d'éléments

JavaScript - DOM Selection 
// Sélection unique
const titre = document.getElementById('titre');
const premier = document.querySelector('.item'); // Premier match

// Sélection multiple (NodeList)
const items = document.querySelectorAll('.item'); // Tous les matchs
const paragraphes = document.getElementsByTagName('p'); // HTMLCollection

// Conversion en tableau pour utiliser les méthodes modernes
const itemsArray = [...document.querySelectorAll('.item')];
// ou
const itemsArray2 = Array.from(document.querySelectorAll('.item'));

5.2 Modification du DOM

JavaScript - DOM Manipulation 
const element = document.querySelector('#monElement');

// Modification du contenu
element.textContent = 'Nouveau texte'; // Texte brut (sécurisé)
element.innerHTML = '<strong>HTML</strong>'; // HTML interprété (attention XSS)

// Attributs
element.setAttribute('data-id', '123');
element.getAttribute('data-id'); // '123'
element.removeAttribute('disabled');

// Classes CSS
element.classList.add('active');
element.classList.remove('hidden');
element.classList.toggle('visible');
element.classList.contains('active'); // true/false

// Styles inline
element.style.backgroundColor = '#f26522';
element.style.cssText = 'color: white; padding: 10px;';

// Création d'éléments
const nouveauDiv = document.createElement('div');
nouveauDiv.className = 'card';
nouveauDiv.textContent = 'Nouveau contenu';
document.body.appendChild(nouveauDiv);

// Insertion avant/après
element.before(nouveauDiv);
element.after(nouveauDiv);
element.replaceWith(nouveauDiv);

5.3 Gestion des événements

JavaScript - Events 
const btn = document.querySelector('#monBouton');

// addEventListener (recommandé)
btn.addEventListener('click', function(event) {
  console.log('Cliqué !');
  console.log(event.target); // Élément cliqué

// Avec fonction fléchée
btn.addEventListener('click', (e) => {

document.querySelector('#liste').addEventListener('click', (e) => {
  if (e.target.matches('.item')) {
    console.log('Item cliqué :', e.target.textContent);
  }
});

// 'input' - Valeur d'input modifiée
// 'change' - Valeur confirmée (blur)
// 'keydown/keyup' - Touches clavier
// 'scroll' - Défilement
// 'DOMContentLoaded' - DOM prêt

6 Programmation Asynchrone

JavaScript est single-threaded mais gère l'asynchronicité via le Event Loop. Comprendre les callbacks, promesses et async/await est essentiel pour les applications modernes.

6.1 Promises

Une Promise représente une valeur qui peut être disponible maintenant, dans le futur, ou jamais :

JavaScript - Promises 
// Création d'une Promise
const fetchData = new Promise((resolve, reject) => {
  setTimeout(() => {
    const success = true;
    if (success) {
      resolve({ data: 'Données reçues' });
    } else {
      reject(new Error('Échec de la requête'));
    }
  }, 1000);
});

// Consommation
fetchData
  .then(result => console.log(result))
  .catch(error => console.error(error))
  .finally(() => console.log('Terminé'));

// Chaînage
fetchUser(1)
  .then(user => fetchOrders(user.id))
  .then(orders => console.log(orders))
  .catch(err => console.error(err));

// Promise.all - attendre plusieurs promesses
Promise.all([fetchUser(), fetchPosts(), fetchComments()])
  .then(([user, posts, comments]) => {
    console.log('Tout est chargé !');
  })
  .catch(err => console.error('Une erreur est survenue'));

// Promise.race - première promesse résolue
Promise.race([fetchFast(), fetchSlow()])
  .then(result => console.log('Le plus rapide :', result));

6.2 Async/Await (ES2017)

La syntaxe async/await rend le code asynchrone aussi lisible que le code synchrone :

JavaScript - Async/Await 
// Fonction async
async function getUserData(userId) {
  try {
    const response = await fetch(`/api/users/${userId}`);

    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`);
    }

    const user = await response.json();
    const orders = await fetch(`/api/orders/${user.id}`).then(r => r.json());

    return { user, orders };
  } catch (error) {
    console.error('Erreur :', error.message);
    throw error; // Re-lancer pour le caller
  }
}

// Appel
getUserData(1)
  .then(data => console.log(data))
  .catch(err => console.error(err));

// Avec Promise.all
async function loadDashboard() {
  const [user, posts, stats] = await Promise.all([
    fetchUser(),
    fetchPosts(),
    fetchStats()
  ]);
  return { user, posts, stats };
}

6.3 Fetch API

L'API Fetch moderne remplace XMLHttpRequest pour les requêtes HTTP :

JavaScript - Fetch API 
// GET request
async function getPosts() {
  const response = await fetch('https://api.example.com/posts');
  if (!response.ok) throw new Error('Erreur réseau');
  return response.json();
}

// POST request
async function createPost(data) {
  const response = await fetch('https://api.example.com/posts', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer token123'
    },
    body: JSON.stringify(data)
  });
  return response.json();
}

// AbortController - annuler une requête
const controller = new AbortController();
fetch('/api/data', { signal: controller.signal });
// Pour annuler : controller.abort();
Gestion d'erreurs

Toujours entourer les appels await dans un bloc try/catch. Les erreurs réseau et les réponses HTTP avec status d'erreur (4xx, 5xx) doivent être gérées explicitement.

7 Modules ES6

Les modules ES6 permettent de diviser le code en fichiers réutilisables avec une portée isolée.

JavaScript - math.js (export) 
// Export nommé
export const PI = 3.14159;
export function addition(a, b) {
  return a + b;
}

// Export par défaut (un seul par fichier)
export default class Calculator {
  constructor() {
    this.result = 0;
  }
  add(n) { this.result += n; return this; }
  getResult() { return this.result; }
}

// Ré-export
export { foo, bar } from './helpers.js';
JavaScript - app.js (import) 
// Import nommé
import { PI, addition } from './math.js';

// Import avec alias
import { addition as add } from './math.js';

// Import tout
import * as Math from './math.js';
Math.addition(2, 3);

// Import par défaut
import Calculator from './math.js';
const calc = new Calculator();

// Import mixte
import Calculator, { PI } from './math.js';

// Import dynamique (lazy loading)
const module = await import('./heavy-module.js');
Architecture modulaire

Organisez votre code en modules par fonctionnalité : api.js pour les requêtes, utils.js pour les fonctions utilitaires, components/ pour les composants UI réutilisables.

8 Stockage Local

8.1 localStorage et sessionStorage

JavaScript - Web Storage 
// localStorage - persiste après fermeture du navigateur
localStorage.setItem('username', 'Alice');
const username = localStorage.getItem('username');
localStorage.removeItem('username');
localStorage.clear(); // Tout supprimer

// Stocker des objets (JSON)
const user = { nom: 'Bob', age: 30 };
localStorage.setItem('user', JSON.stringify(user));
const savedUser = JSON.parse(localStorage.getItem('user'));

// sessionStorage - persiste pendant la session
sessionStorage.setItem('token', 'abc123');

// Vérifier la disponibilité
try {
  localStorage.setItem('test', 'test');
  localStorage.removeItem('test');
} catch (e) {
  console.warn('localStorage non disponible');
}
