1 Introduction au Web

Avant de plonger dans le code, comprenons comment fonctionne le web. Le World Wide Web est un système d'information distribué où les documents sont reliés par des hyperliens et accessibles via Internet.

1.1 Comment fonctionne le web ?

Le web repose sur un modèle client-serveur. Votre navigateur (client) envoie une requête HTTP à un serveur web qui renvoie les fichiers HTML, CSS et JavaScript nécessaires à l'affichage de la page.

À retenir

HTTP (HyperText Transfer Protocol) est le protocole de communication entre le client et le serveur. HTTPS est la version sécurisée qui chiffre les données échangées.

1.2 Les outils du développeur web

Pour commencer, vous avez besoin de :

  • Un éditeur de code : VS Code (recommandé), Sublime Text, ou Atom
  • Un navigateur moderne : Chrome, Firefox, Safari ou Edge
  • Les outils de développement (DevTools) intégrés à votre navigateur

Exercice pratique

Installez VS Code et les extensions suivantes : Live Server, Prettier, et Auto Rename Tag. Ouvrez les DevTools de votre navigateur avec F12 et explorez les onglets Elements, Console et Network.

2 Les bases du HTML

HTML (HyperText Markup Language) est le langage de balisage utilisé pour structurer le contenu des pages web. Il utilise des balises pour définir la signification et la structure du contenu.

2.1 Structure d'un document HTML

Tout document HTML5 suit une structure de base obligatoire :

HTML 
<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Ma première page</title>
</head>
<body>
    <h1>Bonjour le monde !</h1>
    <p>Ceci est mon premier paragraphe.</p>
</body>
</html>

2.2 Les balises essentielles

Voici les balises HTML les plus courantes :

BaliseDescriptionUsage
<h1> à <h6>Titres de niveau 1 à 6Hiérarchie du contenu
<p>ParagrapheTexte courant
<a>Lien hypertexteNavigation
<img>ImageMédia
<ul>, <ol>, <li>ListesÉnumération
<div>Conteneur génériqueDivision
<span>Conteneur en ligneStyle inline

2.3 Les balises sémantiques HTML5

HTML5 introduit des balises sémantiques qui donnent du sens à la structure :

  • <header> : En-tête de page ou de section
  • <nav> : Navigation principale
  • <main> : Contenu principal (unique par page)
  • <article> : Contenu autonome (article, post)
  • <section> : Section thématique
  • <aside> : Contenu connexe (sidebar)
  • <footer> : Pied de page
HTML - Structure sémantique 
<body>
  <header>
    <nav>...</nav>
  </header>

  <main>
    <article>
      <header><h1>Titre</h1></header>
      <section>...</section>
      <section>...</section>
    </article>

    <aside>...</aside>
  </main>

  <footer>...</footer>
</body>
Attention

N'utilisez pas les balises sémantiques uniquement pour leur style par défaut. Utilisez CSS pour le style et HTML pour la structure. Une bonne sémantique améliore l'accessibilité et le SEO.

3 Introduction à CSS

CSS (Cascading Style Sheets) est le langage qui décrit la présentation d'un document HTML. Il contrôle les couleurs, les polices, la mise en page et les animations.

3.1 Syntaxe CSS de base

La syntaxe CSS suit le modèle : sélecteur { propriété: valeur; }

CSS 
/* Sélecteur d'élément */
h1 {
  color: #1a3a6e;
  font-size: 2.5rem;
  text-align: center;
}

/* Sélecteur de classe */
.btn {
  background: #f26522;
  padding: 12px 24px;
  border-radius: 8px;
}

/* Sélecteur d'ID */
#header {
  position: sticky;
  top: 0;
}

3.2 Les sélecteurs CSS

Les sélecteurs permettent de cibler les éléments à styliser :

  • Sélecteur universel * : cible tous les éléments
  • Sélecteur d'élément p : cible tous les paragraphes
  • Sélecteur de classe .class : cible les éléments avec cette classe
  • Sélecteur d'ID #id : cible un élément unique
  • Sélecteur descendant div p : cible les p dans les div
  • Sélecteur enfant direct ul > li : cible les li enfants directs
  • Pseudo-classes :hover, :focus, :nth-child()

3.3 Le modèle de boîte (Box Model)

Chaque élément HTML est une boîte rectangulaire composée de :

  1. Content : le contenu (texte, image)
  2. Padding : marge interne (transparente)
  3. Border : bordure
  4. Margin : marge externe (transparente)
CSS - Box Model 
.box {
  /* Box-sizing pour un calcul plus intuitif */
  box-sizing: border-box;

  width: 300px;
  height: 200px;
  padding: 20px;
  border: 2px solid #1a3a6e;
  margin: 15px auto;

  /* Le total sera bien 300px de large */
}
Bonne pratique

Utilisez toujours box-sizing: border-box; pour tous les éléments. Cela inclut le padding et la border dans le calcul de la largeur/hauteur, ce qui est beaucoup plus intuitif.

4 Flexbox - Mise en page flexible

Flexbox (Flexible Box Layout) est un module CSS conçu pour distribuer l'espace entre les éléments d'un conteneur et contrôler leur alignement, même lorsque leur taille est inconnue ou dynamique.

4.1 Concepts fondamentaux

Flexbox repose sur deux axes :

  • Axe principal (main axis) : défini par flex-direction (row ou column)
  • Axe transversal (cross axis) : perpendiculaire à l'axe principal

4.2 Propriétés du conteneur flex

CSS - Flexbox Container 
.container {
  display: flex;

  /* Direction des éléments */
  flex-direction: row; /* row | row-reverse | column | column-reverse */

  /* Retour à la ligne */
  flex-wrap: wrap; /* nowrap | wrap | wrap-reverse */

  /* Raccourci : direction + wrap */
  flex-flow: row wrap;

  /* Alignement sur l'axe principal */
  justify-content: center; /* flex-start | flex-end | center | space-between | space-around | space-evenly */

  /* Alignement sur l'axe transversal (une ligne) */
  align-items: center; /* stretch | flex-start | flex-end | center | baseline */

  /* Alignement sur l'axe transversal (plusieurs lignes) */
  align-content: space-between; /* stretch | flex-start | flex-end | center | space-between | space-around */
}

4.3 Propriétés des éléments flex

CSS - Flex Items 
.item {
  /* Ordre d'affichage (défaut: 0) */
  order: 1;

  /* Facteur de grossissement */
  flex-grow: 1; /* 0 = ne pas grandir */

  /* Facteur de rétrécissement */
  flex-shrink: 1; /* 0 = ne pas rétrécir */

  /* Taille de base avant distribution */
  flex-basis: 200px;

  /* Raccourci : grow shrink basis */
  flex: 1 1 200px;

  /* Alignement individuel */
  align-self: flex-end; /* auto | flex-start | flex-end | center | baseline | stretch */
}

Projet pratique : Navbar responsive

Créez une barre de navigation avec logo à gauche et liens à droite. En mobile (max-width: 768px), les liens doivent s'empiler verticalement. Utilisez Flexbox pour l'alignement et les media queries pour le responsive.

5 CSS Grid - Grilles avancées

CSS Grid Layout est un système de mise en page bidimensionnel qui permet de créer des grilles complexes avec des lignes et des colonnes. C'est l'outil idéal pour les layouts de page complets.

5.1 Définir une grille

CSS - Grid Basics 
.grid-container {
  display: grid;

  /* Définir 3 colonnes de tailles différentes */
  grid-template-columns: 200px 1fr 200px;

  /* Définir 2 lignes */
  grid-template-rows: 100px auto;

  /* Espacement entre les cellules */
  gap: 20px;

  /* Raccourci pour row-gap + column-gap */
  /* gap: 20px 30px; */
}

5.2 Unités fractionnaires et fonctions

  • fr : unité fractionnaire qui occupe l'espace restant
  • repeat(3, 1fr) : répète 3 colonnes de 1fr
  • minmax(200px, 1fr) : minimum 200px, maximum 1fr
  • auto-fit et auto-fill : ajustement automatique
CSS - Grid Responsive 
/* Grille responsive sans media queries */
.grid-auto {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 20px;
}

/* Placement des éléments */
.header { grid-column: 1 / -1; } /* De la 1ère à la dernière colonne */
.sidebar { grid-row: 2 / 4; }
.footer { grid-column: 1 / -1; }

5.3 Grid vs Flexbox

CritèreFlexboxGrid
DimensionUnidimensionnel (ligne OU colonne)Bidimensionnel (ligne ET colonne)
Cas d'usageComposants, alignement, navbarLayout global de page, galeries
ContenuBasé sur le contenuBasé sur le conteneur
ComplexitéPlus simplePlus puissant
Conseil d'expert

Flexbox et Grid ne sont pas concurrents mais complémentaires. Utilisez Grid pour la structure globale de la page et Flexbox pour l'alignement des éléments internes.

6 Responsive Design

Le responsive design permet d'adapter l'affichage d'un site web à la taille de l'écran de l'utilisateur. C'est aujourd'hui indispensable avec la multiplicité des devices.

6.1 Viewport et media queries

La balise meta viewport est essentielle pour le responsive :

HTML - Viewport 
<meta name="viewport" content="width=device-width, initial-scale=1.0">

6.2 Media queries essentielles

CSS - Media Queries 
/* Mobile first - styles par défaut pour mobile */
.container {
  padding: 15px;
}

/* Tablette */
@media (min-width: 768px) {
  .container {
    padding: 30px;
    max-width: 720px;
    margin: 0 auto;
  }
}

/* Desktop */
@media (min-width: 1024px) {
  .container {
    max-width: 960px;
  }
}

/* Large desktop */
@media (min-width: 1200px) {
  .container {
    max-width: 1140px;
  }
}

6.3 Images responsives

Utilisez max-width: 100% pour que les images ne dépassent jamais leur conteneur :

CSS - Images Responsive 
img {
  max-width: 100%;
  height: auto;
  display: block;
}

/* Images avec différentes résolutions */
<picture>
  <source media="(min-width: 800px)" srcset="image-large.jpg">
  <source media="(min-width: 400px)" srcset="image-medium.jpg">
  <img src="image-small.jpg" alt="Description">
</picture>

Projet final : Landing page responsive

Créez une landing page complète avec : header sticky, hero section, grille de features (3 colonnes desktop, 1 colonne mobile), section testimonials, et footer. Utilisez Grid pour le layout global et Flexbox pour les alignements internes.

7 CSS Avancé

7.1 Variables CSS (Custom Properties)

Les variables CSS permettent de définir des valeurs réutilisables :

CSS - Variables 
:root {
  --primary: #1a3a6e;
  --secondary: #f26522;
  --spacing: 1rem;
  --radius: 8px;
}

.btn {
  background: var(--primary);
  padding: var(--spacing);
  border-radius: var(--radius);
}

/* Avec fallback */
.btn-alt {
  background: var(--primary, blue);
}

7.2 Animations et transitions

CSS - Animations 
/* Transitions simples */
.btn {
  transition: all 0.3s ease;
}
.btn:hover {
  transform: translateY(-3px);
  box-shadow: 0 8px 25px rgba(0,0,0,0.2);
}

/* Keyframes */
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to { opacity: 1; transform: translateY(0); }
}

.element {
  animation: fadeIn 0.6s ease-out forwards;
}

7.3 Pseudo-éléments ::before et ::after

Ces pseudo-éléments permettent d'insérer du contenu décoratif sans HTML supplémentaire :

CSS - Pseudo-éléments 
.quote::before {
  content: '"';
  font-size: 3rem;
  color: var(--secondary);
  line-height: 0;
}

/* Overlay sur image */
.card::after {
  content: '';
  position: absolute;
  inset: 0;
  background: linear-gradient(transparent, rgba(0,0,0,0.7));
  pointer-events: none;
}
