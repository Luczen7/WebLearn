1 Introduction à React

React est une bibliothèque JavaScript développée par Facebook (Meta) pour construire des interfaces utilisateur interactives et réactives. Elle repose sur le concept de composants réutilisables et un DOM virtuel pour des performances optimales.

1.1 Pourquoi React ?

  • Composants réutilisables : UI modulaire et maintenable
  • DOM virtuel : Mises à jour efficaces et performantes
  • Unidirectional data flow : Flux de données prévisible
  • Écosystème riche : Router, state management, testing
  • SSR/SSG : Next.js pour le rendu côté serveur

1.2 Créer un projet React

Bash 
# Avec Vite (recommandé - rapide et moderne)
npm create vite@latest mon-app -- --template react
# ou avec TypeScript
npm create vite@latest mon-app -- --template react-ts

cd mon-app
npm install
npm run dev

# Avec Create React App (legacy)
npx create-react-app mon-app

1.3 Structure d'un projet React

├── public/
│   ├── index.html

├── package.json
└── vite.config.js
Vite vs CRA

Vite est aujourd'hui préféré à Create React App pour sa vitesse de démarrage (10x plus rapide), son hot module replacement instantané, et son support natif de TypeScript et des modules ES.

2 Composants React

Depuis React 16.8 et l'introduction des Hooks, les composants fonctionnels sont la norme :

function Welcome({ name, age }) {
  return (
    <div className="welcome-card">

// Avec arrow function (plus courant)
const UserCard = ({ name, email, avatar, isActive = false }) => {

// Composition de composants
const App = () => {

3 React Hooks Essentiels

Les Hooks permettent d'utiliser l'état et d'autres fonctionnalités React dans les composants fonctionnels.

3.1 useState - Gestion de l'état local

React - useState 
import { useState } from 'react';

const Counter = () => {
  const [count, setCount] = useState(0);
  const [user, setUser] = useState({ name: '', email: '' });

  const increment = () => setCount(prev => prev + 1);
  const decrement = () => setCount(prev => prev - 1);

  const updateName = (name) => {
    setUser(prev => ({ ...prev, name })); // Spread pour préserver les autres champs
  };

  return (
    <div>
      <p>Compteur : {count}</p>
      <button onClick={decrement}>-</button>
      <button onClick={increment}>+</button>
    </div>
  );
};

// useState avec fonction d'initialisation (lazy)
const ExpensiveComponent = () => {
  const [data, setData] = useState(() => {
    // Cette fonction ne s'exécute qu'au premier rendu
    return computeExpensiveValue();
  });
};

3.2 useEffect - Effets de bord

React - useEffect 
import { useState, useEffect } from 'react';

const UserProfile = ({ userId }) => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  // Exécuté après chaque rendu (éviter !)
  useEffect(() => {
    console.log('Rendu effectué');
  });

  // Exécuté une seule fois au montage
  useEffect(() => {
    console.log('Composant monté');
    return () => {
      console.log('Composant démonté (cleanup)');
    };
  }, []);

  // Exécuté quand userId change
  useEffect(() => {
    const fetchUser = async () => {
      setLoading(true);
      try {
        const response = await fetch(`/api/users/${userId}`);
        const data = await response.json();
        setUser(data);
      } catch (error) {
        console.error(error);
      } finally {
        setLoading(false);
      }
    };

    fetchUser();
  }, [userId]); // Dépendances

  if (loading) return <Spinner />;
  return <div>{user.name}</div>;
};

3.3 useRef et useCallback

React - useRef & useCallback 
import { useRef, useCallback } from 'react';

// useRef - Référence mutable qui ne déclenche pas de re-render
const TextInput = () => {
  const inputRef = useRef(null);
  const renderCount = useRef(0);

  renderCount.current++;

  const focusInput = () => {
    inputRef.current.focus();
  };

  return (
    <div>
      <input ref={inputRef} type="text" />
      <button onClick={focusInput}>Focus</button>
      <p>Rendus : {renderCount.current}</p>
    </div>
  );
};

// useCallback - Mémoriser une fonction
const Parent = () => {
  const [count, setCount] = useState(0);

  // Sans useCallback, cette fonction est recréée à chaque rendu
  // et le composant enfant est re-render inutilement
  const handleClick = useCallback(() => {
    console.log('Cliqué');
  }, []); // [] = jamais recréée

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(c => c + 1)}>+</button>
      <ChildComponent onClick={handleClick} />
    </div>
  );
};

3.4 useMemo - Mémorisation de valeurs

React - useMemo 
import { useMemo } from 'react';

const ProductList = ({ products, filter }) => {
  // Sans useMemo, filteredProducts est recalculé à chaque rendu
  const filteredProducts = useMemo(() => {
    console.log('Filtrage exécuté');
    return products
      .filter(p => p.name.toLowerCase().includes(filter.toLowerCase()))
      .sort((a, b) => a.price - b.price);
  }, [products, filter]); // Recalculé seulement si products ou filter change

  return (
    <ul>
      {filteredProducts.map(product => (
        <ProductItem key={product.id} product={product} />
      ))}
    </ul>
  );
};
Attention

N'abusez pas de useMemo et useCallback. Ils ont un coût (comparaison des dépendances). Utilisez-les uniquement pour des calculs coûteux ou quand un composant enfant est optimisé avec React.memo.

4 React Router

React Router est la bibliothèque de routing standard pour React, permettant la navigation entre les pages sans rechargement.

React - React Router v6 
import { 
  BrowserRouter, 
  Routes, 
  Route, 
  Link, 
  useParams,
  useNavigate,
  Outlet
} from 'react-router-dom';

// Configuration des routes
const App = () => {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Accueil</Link>
        <Link to="/products">Produits</Link>
        <Link to="/about">A Propos</Link>
      </nav>

      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/products" element={<ProductLayout />}>
          <Route index element={<ProductList />} />
          <Route path=":id" element={<ProductDetail />} />
          <Route path="new" element={<ProductForm />} />
        </Route>
        <Route path="/about" element={<About />} />
        <Route path="*" element={<NotFound />} />
      </Routes>
    </BrowserRouter>
  );
};

// Layout avec Outlet pour les routes imbriquées
const ProductLayout = () => {
  return (
    <div>
      <h1>Produits</h1>
      <Outlet /> {/* Les routes enfants s'affichent ici */}
    </div>
  );
};

// Paramètres d'URL
const ProductDetail = () => {
  const { id } = useParams();
  const navigate = useNavigate();

  return (
    <div>
      <h2>Produit #{id}</h2>
      <button onClick={() => navigate(-1)}>Retour</button>
      <button onClick={() => navigate('/products')}>Tous les produits</button>
    </div>
  );
};

5 Gestion d'État avec Redux Toolkit

Redux Toolkit est la façon moderne et recommandée d'utiliser Redux. Il réduit le boilerplate et intègre les bonnes pratiques.

Redux Toolkit - Store 
import { configureStore, createSlice } from '@reduxjs/toolkit';

// Création d'un slice (état + reducers + actions)
const counterSlice = createSlice({
  name: 'counter',
  initialState: { value: 0 },
  reducers: {
    increment: (state) => {
      state.value += 1; // Immer permet la mutation !
    },
    decrement: (state) => {
      state.value -= 1;
    },
    incrementByAmount: (state, action) => {
      state.value += action.payload;
    },
  },
});

// Export des actions
export const { increment, decrement, incrementByAmount } = counterSlice.actions;

// Configuration du store
export const store = configureStore({
  reducer: {
    counter: counterSlice.reducer,
    // autres slices...
  },
});

// Types TypeScript
export type RootState = ReturnType<typeof store.getState>;
export type AppDispatch = typeof store.dispatch;
React - Utilisation Redux 
import { Provider, useSelector, useDispatch } from 'react-redux';
import { store, increment, decrement } from './store';

// Provider dans App.jsx
const App = () => (
  <Provider store={store}>
    <Counter />
  </Provider>
);

// Composant connecté
const Counter = () => {
  const count = useSelector((state) => state.counter.value);
  const dispatch = useDispatch();

  return (
    <div>
      <span>{count}</span>
      <button onClick={() => dispatch(increment())}>+</button>
      <button onClick={() => dispatch(decrement())}>-</button>
    </div>
  );
};

// Async thunk pour les appels API
import { createAsyncThunk } from '@reduxjs/toolkit';

export const fetchUsers = createAsyncThunk(
  'users/fetchUsers',
  async (_, { rejectWithValue }) => {
    try {
      const response = await fetch('/api/users');
      if (!response.ok) throw new Error('Erreur serveur');
      return await response.json();
    } catch (error) {
      return rejectWithValue(error.message);
    }
  }
);

6 Next.js - React en Production

Next.js est le framework React le plus populaire, offrant le rendu côté serveur (SSR), la génération statique (SSG), et un routing basé sur le filesystem.

Next.js - App Router 
// app/page.tsx - Page d'accueil
async function HomePage() {
  const posts = await fetch('https://api.example.com/posts', {
    next: { revalidate: 3600 } // ISR : revalidation toutes les heures
  }).then(r => r.json());

  return (
    <main>
      <h1>Blog</h1>
      <div className="grid">
        {posts.map(post => (
          <PostCard key={post.id} post={post} />
        ))}
      </div>
    </main>
  );
}

// app/posts/[id]/page.tsx - Route dynamique
async function PostPage({ params }: { params: { id: string } }) {
  const post = await fetch(`https://api.example.com/posts/${params.id}`)
    .then(r => r.json());

  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}

// app/layout.tsx - Layout racine
export default function RootLayout({ children }) {
  return (
    <html lang="fr">
      <body>
        <Header />
        {children}
        <Footer />
      </body>
    </html>
  );
}

Stratégies de rendu Next.js

  • SSR (Server-Side Rendering) : Rendu à chaque requête, données toujours fraîches
  • SSG (Static Site Generation) : Rendu au build, performance maximale
  • ISR (Incremental Static Regeneration) : SSG avec revalidation périodique
  • CSR (Client-Side Rendering) : Rendu dans le navigateur, pour les données utilisateur

7 Performance et Optimisation

7.1 React.memo

React.memo empêche le re-render d'un composant si ses props n'ont pas changé :

React - Optimisation 
import { memo, useMemo, useCallback } from 'react';

// Composant mémorisé
const ExpensiveList = memo(({ items, onSelect }) => {
  console.log('ExpensiveList rendu');
  return (
    <ul>
      {items.map(item => (
        <li key={item.id} onClick={() => onSelect(item.id)}>
          {item.name}
        </li>
      ))}
    </ul>
  );
});

// Avec comparaison personnalisée
const CustomComparison = memo(({ user }) => {
  return <div>{user.name}</div>;
}, (prevProps, nextProps) => {
  return prevProps.user.id === nextProps.user.id;
});

// Lazy loading de composants
import { lazy, Suspense } from 'react';
const HeavyComponent = lazy(() => import('./HeavyComponent'));

const App = () => (
  <Suspense fallback={<Spinner />}>
    <HeavyComponent />
  </Suspense>
);
