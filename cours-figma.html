1 Introduction à Figma et au Design UI/UX

Figma est un outil de design d'interfaces basé sur le cloud qui permet de créer des maquettes, des prototypes et des design systems en collaboration temps réel. C'est devenu l'outil standard de l'industrie.

1.1 Pourquoi Figma ?

  • Cloud-based : Travaillez de n'importe où, sur n'importe quel OS
  • Collaboration temps réel : Plusieurs designers sur le même fichier
  • Prototypage intégré : Créez des interactions sans quitter l'outil
  • Design system natif : Composants, variables, et styles partagés
  • Plugins riches : Écosystème extensible
  • Dev Mode : Handoff facile aux développeurs

1.2 Interface et outils de base

L'interface Figma se compose de plusieurs zones :

  • Toolbar (haut) : Outils de sélection, formes, texte, pen
  • Layers panel (gauche) : Hiérarchie des éléments
  • Canvas (centre) : Zone de travail
  • Properties panel (droite) : Propriétés de l'élément sélectionné
  • Assets panel : Composants et styles réutilisables

2 Wireframing et Architecture de l'Information

2.1 Qu'est-ce qu'un wireframe ?

Un wireframe est une représentation schématique basse fidélité d'une interface. Il se concentre sur :

  • La structure : Organisation des éléments sur la page
  • La hiérarchie : Importance relative des éléments
  • La navigation : Flux utilisateur entre les pages
  • Le contenu : Placement du texte et des images

Il ne se concentre PAS sur : les couleurs, les polices, les images réelles, les animations.

2.3 Créer des wireframes dans Figma

Figma - Wireframe Setup 
// Bonnes pratiques wireframing dans Figma :

1. Utiliser des couleurs neutres (gris #D9D9D9, #B3B3B3, #808080)
2. Utiliser des placeholders pour les images (rectangle + icône image)
3. Utiliser du faux texte (Lorem ipsum ou plugins comme "Lorem Ipsum")
4. Créer des composants réutilisables pour les éléments communs :
   - Header wireframe
   - Footer wireframe  
   - Card wireframe
   - Formulaire wireframe

5. Organiser en pages :
   - Page "Wireframes Desktop"
   - Page "Wireframes Mobile"
   - Page "User Flow"

6. Nommer les frames selon le format : [Numéro] - [Nom de la page]
   Ex: 01 - Homepage, 02 - Product List, 03 - Product Detail

Exercice : Wireframe E-commerce

Créez les wireframes pour un site e-commerce comprenant : Homepage (header, hero, catégories, produits populaires, footer), Page produit (images, infos, avis, produits similaires), Panier, et Checkout (formulaire, récapitulatif, paiement). Utilisez uniquement des rectangles gris et du texte.

3 Design d'Interfaces (UI Design)

3.1 Principes fondamentaux du design visuel

  • Hiérarchie visuelle : Guider l'œil de l'utilisateur (taille, couleur, contraste, espacement)
  • Alignement : Créer de l'ordre et de la cohérence
  • Contraste : Différencier les éléments importants
  • Répétition : Cohérence à travers l'interface
  • Proximité : Regrouper les éléments liés
  • Équilibre : Distribution visuelle harmonieuse

3.2 Système de couleurs

Un système de couleurs cohérent comprend :

  • Couleur primaire : Actions principales, éléments clés
  • Couleur secondaire : Actions secondaires, accents
  • Couleurs sémantiques : Succès (vert), erreur (rouge), avertissement (jaune), info (bleu)
  • Neutres : Texte, fonds, bordures (nuances de gris)

3.3 Typographie

Règles typographiques essentielles :

  • Maximum 2-3 polices par projet
  • Échelle typographique : Définir des tailles cohérentes (12, 14, 16, 18, 20, 24, 32, 48px)
  • Interligne : 1.5x la taille de police pour le corps de texte
  • Longueur de ligne : 45-75 caractères maximum
  • Contraste : Ratio minimum 4.5:1 pour l'accessibilité WCAG AA

3.4 Espacement et grille

Design - Système d'espacement 8pt 
// Système d'espacement basé sur 8px :

4px   - Très petit (icônes proches du texte)
8px   - Petit (padding interne boutons)
16px  - Standard (gap entre éléments, padding cards)
24px  - Moyen (espacement sections)
32px  - Grand (espacement blocs)
48px  - Très grand (espacement sections majeures)
64px  - Section (espacement entre sections de page)

// Grille de colonnes
Desktop : 12 colonnes, gutter 24px, margin 48px
Tablet  : 8 colonnes, gutter 24px, margin 32px
Mobile  : 4 colonnes, gutter 16px, margin 16px

// Dans Figma : Layout Grid > Columns
// Columns: 12, Type: Stretch, Gutter: 24, Margin: 48

4 Auto Layout et Composants

L'Auto Layout est la fonctionnalité la plus puissante de Figma. Il permet de créer des designs responsives et des composants intelligents qui s'adaptent automatiquement à leur contenu.

4.1 Fonctionnement de l'Auto Layout

Figma - Auto Layout 
// Auto Layout = Flexbox visuel

// Direction :
- Vertical (column) : Éléments empilés verticalement
- Horizontal (row) : Éléments côte à côte

// Propriétés essentielles :
- Spacing between items : Espacement entre les enfants
- Padding : Marge interne (top, right, bottom, left)
- Alignment : Alignement des enfants (start, center, end, space-between)

// Distribution :
- Hug contents : Le parent s'adapte à la taille des enfants
- Fixed width/height : Taille fixe
- Fill container : Remplit l'espace disponible

// Exemples d'usage :
1. Bouton : Texte + Auto Layout (hug, padding 16px 24px)
2. Card : Image + Texte + Auto Layout vertical
3. Navbar : Logo + Liens + Auto Layout horizontal
4. Liste : Plusieurs items en Auto Layout vertical
5. Formulaire : Label + Input + Error en Auto Layout vertical

4.2 Composants et variants

Figma - Composants & Variants 
// Créer un composant : Ctrl/Cmd + Alt + K
// Créer une instance : Copier-coller le composant

// Structure d'un composant bouton avec variants :

Composant : Button
Variants :
  - Type : Primary / Secondary / Ghost / Danger
  - Size : Small / Medium / Large
  - State : Default / Hover / Active / Disabled / Loading
  - Icon : Left / Right / None

// = 4 × 3 × 5 × 3 = 180 combinaisons possibles !

// Bonnes pratiques variants :
1. Nommer les propriétés en camelCase : "type", "size", "state"
2. Nommer les valeurs en PascalCase : "Primary", "Medium", "Default"
3. Utiliser des couleurs de fond différentes pour chaque frame variant
4. Documenter le composant avec des notes

// Properties (Figma 2022+) :
- Text properties : Texte modifiable dans l'instance
- Boolean properties : Afficher/masquer des éléments
- Instance swap : Remplacer un nested component
- Variant properties : Changer de variant
Astuce pro

Utilisez les Component Properties (text, boolean, instance swap) pour réduire le nombre de variants nécessaires. Un bouton avec icône optionnelle n'a plus besoin de variants "with icon" / "without icon" - utilisez une propriété boolean.

5 Prototypage Interactif

Le prototypage permet de simuler l'expérience utilisateur réelle avant le développement. Figma offre des interactions riches pour tester vos designs.

5.1 Types d'interactions

  • On Click : Navigation vers une autre frame
  • On Hover : Changement d'état au survol
  • On Drag : Glisser-déposer, carrousels
  • After Delay : Animation automatique après un délai
  • On Scroll : Parallax, sticky headers
  • Smart Animate : Transition fluide entre deux états

5.2 Smart Animate

Smart Animate détecte automatiquement les différences entre deux frames et anime la transition :

  • Position (x, y)
  • Dimension (width, height)
  • Opacité
  • Rotation
  • Couleur de fond
  • Rayon de bordure
Figma - Prototype Flow 
// Exemple de flow utilisateur E-commerce :

Frame 1: Homepage
  - "Voir produits" → On Click → Navigate to → Frame 2
  - "Panier" → On Click → Open overlay → Frame 5

Frame 2: Product List
  - Filtre "Prix" → On Click → Smart Animate → Frame 3 (filtres ouverts)
  - Produit 1 → On Click → Navigate to → Frame 4

Frame 3: Product List (filtres ouverts)
  - "Appliquer" → On Click → Smart Animate → Frame 2

Frame 4: Product Detail
  - "Ajouter au panier" → On Click → Smart Animate → Frame 4 (état ajouté)
  - "Continuer" → On Click → Navigate to → Frame 5

Frame 5: Cart Overlay
  - "Commander" → On Click → Navigate to → Frame 6

Frame 6: Checkout
  - "Payer" → On Click → After Delay 2s → Frame 7

Frame 7: Confirmation
  - Animation : Scale up + Fade in

// Réglages de transition :
- Animation : Smart Animate (Ease Out, 300ms)
- Ou : Instant, Dissolve, Move, Slide, Push

Projet : Prototype App Mobile

Créez un prototype complet d'une application mobile de livraison de nourriture avec : écran d'accueil avec animations, liste des restaurants avec filtres, détail restaurant avec menu, panier avec animations d'ajout, checkout avec formulaires, et confirmation avec animation de succès. Utilisez Smart Animate pour toutes les transitions.

6.2 Variables Figma (v2023)

Les Variables sont le successeur des Styles, permettant des tokens de design plus puissants :

  • Color variables : Couleurs réutilisables avec modes (light/dark)
  • Number variables : Espacements, rayons, tailles
  • String variables : Texte réutilisable
  • Boolean variables : Afficher/masquer conditionnellement

7 Dev Mode et Handoff

Le Dev Mode (anciennement Inspect) permet aux développeurs d'accéder facilement aux spécifications de design.

7.1 Préparer un fichier pour les développeurs

  • Nommer les layers de manière sémantique (pas "Rectangle 47")
  • Utiliser des frames au lieu de groupes quand possible
  • Documenter les comportements responsives
  • Indiquer les états interactifs (hover, active, disabled)
  • Ajouter des annotations pour les comportements complexes
  • Organiser les pages : Design, Prototype, Archive

7.2 Export d'assets

Figma - Export 
// Formats d'export recommandés :

// Images (photos, illustrations) :
- Format : JPG (qualité 80%)
- 1x et 2x pour le retina

// Icônes et logos :
- Format : SVG (vectoriel, léger)
- Ou PNG avec fond transparent

// Composants UI :
- Format : SVG pour les formes simples
- Format : PNG pour les effets complexes (ombres, dégradés)

// Dans Figma :
1. Sélectionner l'élément
2. Cliquer sur "Export [nom]" dans le panel droit
3. Choisir le format et le suffixe
4. Cliquer sur l'icône d'export

8 Conclusion

Vous maîtrisez maintenant Figma pour créer des designs professionnels. Prochaines étapes :

  • Plugins avancés : Tokens Studio, Autoflow, Unsplash
  • Design tokens : Intégration avec le code via Style Dictionary
  • Micro-interactions : Animations avancées avec After Effects ou Rive
  • Tests utilisateurs : Maze, Useberry pour tester vos prototypes
  • Design ops : Organisation et scaling du design system
